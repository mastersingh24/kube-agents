# Multi-Cluster GKE Staging PoC (Workloads + Simulators)

This repository templates a dynamic, multi-cluster GKE staging environment. It sets up arbitrary lists of GKE Autopilot and GKE Standard clusters, deploying a modular bundle of workloads (including web tiers, databases, and AI/ML gateways) and traffic simulators to test routing, latency, and resource configurations.

---

## Architecture Overview

- **GKE Autopilot Clusters:** Map-configured in `variables.tf`.
- **GKE Standard Clusters:** Map-configured in `variables.tf`.
- **Workload Bundle:**
  - **Stateless Tier:** Go, Node.js, Python, and Nginx web servers.
  - **Stateful Tier:** Redis Cache, PostgreSQL Database, MongoDB Database.
  - **AI/ML specialized:** LiteLLM Gateway, Langflow, Jupyter Notebook, PyTorch Training.
  - **Traffic Simulators:** K6 HTTP load simulator, Custom TCP/DB connection verify daemon.

## Configuration Setup (Required)

Before running the deployment, you must specify your target GCP Project ID. You can also customize or override the default cluster maps.

### Method 1: Local TFVars (Recommended)

Create a file named `terraform.tfvars` in this directory:

```hcl
project_id = "your-gcp-project-id"

# (Optional) Override GKE Autopilot clusters map
autopilot_clusters = {
  "wolfram-auto-01" = {
    location      = "us-east1"
    traffic_shape = "Steady"
  }
  "wolfram-auto-02" = {
    location      = "asia-east1"
    traffic_shape = "Spikes"
    enable_ray    = true
  }
}

# (Optional) Override GKE Standard clusters map
standard_clusters = {
  "wolfram-std-01" = {
    location       = "us-east1"
    node_locations = ["us-east1-b"]
    traffic_shape  = "Steady"
  }
}
```

_(Note: `*.tfvars` files are local-only and ignored by git)._

### Method 2: Environment Variables

Alternatively, you can set it directly in your terminal:

```bash
export TF_VAR_project_id="your-gcp-project-id"
```

If you attempt to deploy without setting the project ID, Terraform will fail immediately with an input validation error.

## Operations Guide

### 🚀 Deploying the Stack

Run the deploy script. This will initialize Terraform, provision both clusters, configure local `kubectl` contexts, and deploy the workload bundle via Helm:

```bash
./deploy_infra.sh
```

### 📊 Monitoring Logs

Once deployed, you can monitor the HTTP request logs generated by the K6 load simulator on either cluster:

- **For GKE Autopilot:**
  ```bash
  gcloud container clusters get-credentials wolfram-auto-01 --region us-east1 --project <project_id>
  kubectl logs -n staging-workloads -l app=traffic-simulator --tail=50 -f
  ```
- **For GKE Standard:**
  ```bash
  gcloud container clusters get-credentials wolfram-std-01 --region us-east1 --project <project_id>
  kubectl logs -n staging-workloads -l app=traffic-simulator --tail=50 -f
  ```

### 🗑️ Tearing Down the Stack

To uninstall all Helm workloads and completely destroy the Terraform-created GKE clusters and associated networking:

```bash
./teardown_infra.sh
```
